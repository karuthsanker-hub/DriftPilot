## Incidents (real bugs caught at runtime)

10 distinct bugs surfaced today, all caught and fixed live, all preserved in PRs #18 and #19:

1. Operator wasn't configuring root logger → INFO logs silent
2. `AlpacaBrokerClient` missing `reconcile_open_positions` (BrokerReconciler protocol)
3. `LiveAlpacaPositionMonitor.monitor()` method name mismatch
4. Signal registry can't construct catalyst signals (need bus injection)
5. `_ScanResult` shim missing `spy_bar_at` for REGIME_CHECK
6. **Classifier too strict** for live news ("Q1 Earnings Call" wasn't matching) — loosened
7. **No DB bootstrap** for signal `_active_events` — added
8. **Datetime in metadata** broke JSON serialization
9. **Broker race on entry** — orders fill faster than `_wait_for_fill` polls; added Alpaca position reconcile
10. **Broker race on exit** — same shape; without fix, second exit cycle SOLD SHORT (caught and bought-to-cover within 60s)

Plus operational tweaks: max_event_age_minutes 60→240, max_trades_per_symbol_per_day 3→1.

## Known issues to fix tomorrow

1. **Local realized P&L uses computed mid not actual fill price.** Today's local DB had MXL at -$73 (using bad mid) when real fill was -$2.73. Patched manually post-hoc; needs proper fix querying Alpaca `filled_avg_price` after exit submission.
2. **Audit script should query Alpaca for ground-truth fills** when local realized differs from broker-confirmed.
3. **Wide-spread quote filter.** BTSG had $52.94/$60.96 bid/ask. The mid is misleading. Add filter: skip candidate if `(ask-bid)/bid > 5%`.
4. **Manually-reconciled positions lose catalyst metadata** because they go through the post-hoc reconciler path, not the original allocator. Fix: have reconciler look up the slot's stored candidate metadata too.
5. **Re-validate the loosened classifier** via spike on 2024 data with the new patterns. Today's success doesn't prove the validated edge transfers to the modified classifier.
6. **Default operator log level** should be INFO when `--paper-live` (currently silent without flag).
